# Vestige Platform – Frontend

## Overview

**Vestige** is a fashion marketplace platform focused on **avant-garde fashion products**, blending commerce with a strong editorial and artistic identity. The platform is designed to support buying and selling fashion items while emphasizing visual storytelling and a bold aesthetic inspired by **fashion editorials** and **punk rock culture**.

This repository contains the **Frontend architecture and implementation**, where I was responsible for **overall FE architecture, feature organization, and state/data handling strategy**.

---

## Key Responsibilities

* Designed and maintained the **Frontend architecture** following the **Feature-First** approach
* Implemented scalable UI components and layout system
* Managed global and server state efficiently
* Ensured consistency between design, UX, and technical implementation

---

## Tech Stack

### Core

* **Next.js** – App Router architecture
* **TypeScript** – Type-safe development

### Styling & UI

* **Tailwind CSS** – Utility-first styling
* **shadcn/ui** – Accessible and reusable UI components

### State & Data Management

* **TanStack Query (React Query)** – Server state, caching, and data synchronization
* **Zustand** – Lightweight global state management (UI state, local interactions)


---

## Design Philosophy

Vestige’s UI combines:

* **Avant-garde fashion aesthetics**
* **Editorial-style layouts** (bold typography, asymmetry, strong visual hierarchy)
* **Punk rock influences** for attitude and contrast

The goal is not just usability, but **expressing fashion identity through interface design**.

---

## Performance & Best Practices

* Optimized component rendering
* Reusable and composable UI components
* Strict typing for API responses
* Clean separation between UI, logic, and data layers
